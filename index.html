企业级 Agent 协作与调度平台

架构负责人 · Agent 工程 + 全栈 在建 · Flagship

面向组织协作、任务派发、员工数字分身和管理驾驶舱等场景，构建企业级 Agent 调度平台：由 SuperAgent 完成理解、路由、拆解、协商、执行与结果回流。

  • SuperAgent 路由与任务编排
  • A2A 协作与人机混合调度
  • Human-in-the-loop + 实时事件回流
FastAPIReactTypeScript AgentScopeA2A / A2UISSEMySQL
展开背景 · 难点 · 方案 · 结果

业务背景

组织协作中讨论频繁发生，但任务沉淀、执行派发、状态追踪成本高；AI 如果停留在独立聊天框，就无法进入真实协作流程。

技术难点

如何把人、数字分身、业务 Agent 和系统能力纳入统一调度；如何在路由不确定、工具调用失败、多人协商和关键动作确认之间保持可控、可解释、可恢复。

解决方案

设计 SuperAgent 调度层 + Agent Runtime + A2A/A2UI 协议 + Domain EventBus 的分层架构，把意图理解、候选路由、任务草稿、协商确认和执行回流串成闭环。

阶段结果

形成 Human-in-the-loop 的可控执行机制：AI 生成任务草稿和人员建议，人确认后进入执行追踪，任务、协商和建议通过事件流实时回到前端。

我具体做了什么

负责从 0 到 1 梳理产品和架构闭环：定义 SuperAgent、业务 Agent、数字分身和人工确认之间的职责边界；设计任务草稿、协商执行、上下文装配、结果汇总和事件回流链路，确保 AI 行为可审查、关键动作可确认、执行过程可追踪。

展开背景 · 难点 · 方案 · 结果

业务背景

平台需要承载多种金融产品形态的交易、账户与清算，且对资金安全与稳定性要求极高。

技术难点

多产品类型的账户体系与清算核心；高压场景下的稳定性；存量系统重构不影响线上。

解决方案

主导核心系统架构设计，搭建统一运营平台，推进账户体系与清算核心系统重构，落地高可用方案与标准化代码规范。

量化结果

支撑平台核心资产链路稳定运行；建立标准化架构与代码规范、统一技术框架，降低新人上手成本、提升团队交付一致性。

我具体做了什么

把交易、账户、清算、运营之间的关系重新拆成清晰模块，先处理账户模型、清算状态流转、异常补偿和对账边界，再制定灰度迁移方案，避免重构直接冲击线上核心链路。

智能投融资分析系统

AI 工作流 · 架构 金融科技平台

把 LLM 以可追踪工作流接入投研与投后管理流程，让 AI 参与材料读取、信息抽取、结构化分析、异常归因和人工复核。

  • LangGraph 节点化工作流
  • 资料提取与结构化分析
  • 关键结论人工复核与异常跟进
PythonLangGraphLangChainLLMRAG
展开背景 · 难点 · 方案 · 结果

业务背景

投研与投后管理依赖大量人工分析与异常排查，效率与一致性受限。

技术难点

把 LLM 能力可靠地接入投研流程，既要拆解材料、保留依据，又不能让模型直接替代业务判断；投后异常还需要可视化识别与跟进。

解决方案

用 LangGraph 把“读材料、抽信息、做判断、给结论”拆成多个节点，每一步定义输入、输出、失败兜底和人工复核口径，将模型输出纳入可追踪流程。

结果

投研分析从人工整理走向流程化辅助；投后异常从零散排查转为可视化识别与跟进。

我具体做了什么

负责把投研/投后流程抽象成可编排工作流：定义材料输入、节点产物、结构化字段、异常归因和人工复核边界，让 AI 输出可以被业务人员校验、追踪和继续处理。
